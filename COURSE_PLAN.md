# 近期課程規劃

本文件記錄 L3 到 L10 的課程順序與題型定位。這份規劃用於課程設計，不是學生網站頁面。

## 規劃原則

- 每課以一種常見解題型態為核心，不只列語法主題。
- 先讓學生會使用工具解題，再補效率分析與演算法原理。
- 排序演算法原理不放在排序工具之前；前期先學會使用 `sort`。
- 遞迴放在已有函式、序列處理與複雜度概念之後，降低抽象負擔。
- Greedy 不塞進 L3，只在排序題中輕微接觸「排序後較容易選擇」的想法。

## 建議順序

```text
L3：排序與資料整理
L4：字串、字元與函式
L5：時間複雜度與暴力解的界線
L6：prefix sum 與區間查詢
L7：遞迴入門
L8：binary search
L9：greedy 入門
L10：排序演算法概念與 divide and conquer
```

## L3：排序與資料整理

題型重點：把資料排序後，讓比較、選擇、檢查變簡單。

主要內容：

- `sort(a.begin(), a.end())`
- 升冪與降冪排序
- 排序後取最大、最小、前幾大、前幾小
- 排序後檢查相鄰元素
- 簡單去重與統計不同值數量
- 輕量的排序後模擬

適合題型：

- 給一串數字，排序後找第 $K$ 大或第 $K$ 小。
- 排序後檢查是否有重複。
- 排序後找相鄰差的最小值。
- 排序後統計不同數字有幾種。
- 排序後依序處理物品或事件。

不建議放入：

- Bubble sort、merge sort、quicksort 等排序演算法實作。
- 正式 greedy 證明。

## L4：字串、字元與函式

題型重點：處理文字格式、逐字元分析，並開始把重複邏輯拆成 function。

主要內容：

- `string` 與 `char`
- `s[i]` 與 `s.size()`
- 字元比較、大小寫、數字字元
- 字串反轉與回文判斷
- 簡單 function
- 把重複判斷邏輯包成 function

適合題型：

- 判斷一個字串是否為 palindrome。
- 統計字元出現次數。
- 判斷字串是否符合格式。
- 寫 `isPrime`、`isPalindrome`、`digitSum` 等簡單函式。
- 多筆輸入中重複使用同一段判斷邏輯。

## L5：時間複雜度與暴力解的界線

題型重點：從題目限制判斷暴力解是否可行，理解為什麼需要更快的方法。

主要內容：

- 從 $N$ 的大小估算可接受作法
- $O(N)$、$O(N^2)$、$O(N^3)$、$O(N \log N)$ 的直覺
- 暴力枚舉 pair / triple
- 用計數陣列降低查找成本
- 比較「每次重新算」與「先整理資料」

適合題型：

- $N \le 100$ 時枚舉所有 pair 或 triple。
- 判斷兩數和是否等於目標值。
- 統計有幾對數字相同。
- 根據限制選擇 $O(N^2)$ 或 $O(N \log N)$ 作法。
- 分析一段程式大約會跑幾次。

## L6：prefix sum 與區間查詢

題型重點：多次詢問區間資訊時，不能每次重新計算。

主要內容：

- `prefix[i]` 表示前 $i$ 個元素的總和
- 區間和 `prefix[r] - prefix[l - 1]`
- `0`-based 與 `1`-based 寫法
- 多筆查詢
- 計數型 prefix sum
- 固定長度區間總和

適合題型：

- 給陣列，回答多次 $[L, R]$ 區間總和。
- 統計區間內偶數、奇數或特定類別數量。
- 找長度 $K$ 的連續區間最大總和。
- 從 $O(NQ)$ 改成 $O(N + Q)$。

可放進 challenge：

- 簡單差分。
- 二維 prefix sum 的直覺題。

## L7：遞迴入門

題型重點：把問題拆成「目前一步」與「剩下交給遞迴」。

主要內容：

- Function call 的概念
- Base case
- Recursive case
- 遞迴計算
- 遞迴產生所有可能
- 簡單 backtracking

適合題型：

- 遞迴計算 factorial、總和、GCD。
- 印出所有長度 $N$ 的 binary strings。
- 從 $N$ 個物品中選或不選。
- 產生簡單排列或組合。
- 用遞迴枚舉所有可能答案。

不建議放入：

- 正式 graph DFS。
- 複雜 backtracking 剪枝。
- Dynamic programming。

## L8：binary search

題型重點：在有序資料或單調條件中快速找位置或答案。

主要內容：

- Sorted array 中找數字
- 找第一個大於等於 $X$ 的位置
- `lower_bound` / `upper_bound`
- 統計小於等於 $X$ 的數量
- 手寫 binary search
- Answer binary search 的初步概念

適合題型：

- 查詢某個數是否存在。
- 查詢插入位置或排名。
- 統計排序後陣列中落在 $[L, R]$ 的數量。
- 給一個可行性判斷，找最小可行答案。

可放進 challenge：

- 簡單 answer binary search。

## L9：greedy 入門

題型重點：每一步做局部選擇，並能說明為什麼這樣不會讓答案變差。

主要內容：

- 排序後從小到大選
- 排序後從結束時間早的事件開始選
- 配對問題
- 資源有限時最大化數量
- 用反例檢查錯誤策略
- 非形式化 exchange argument

適合題型：

- 用有限預算買最多物品。
- 最多能參加幾個不重疊活動。
- 兩兩配對後最佳化最大值或最小值。
- 排序後依序選擇可行項目。

不建議放入：

- 需要嚴格證明的進階 greedy。
- Huffman coding。
- MST。

## L10：排序演算法概念與 divide and conquer

題型重點：理解排序成本與分治，不是為了取代 `sort`。

主要內容：

- Bubble sort、selection sort、insertion sort 的概念
- $O(N^2)$ 排序與 $O(N \log N)$ 排序的差距
- Merge sort 與遞迴分治
- Quicksort 概念
- Divide and conquer 的基本想法

適合題型：

- 模擬排序過程。
- 計算簡單交換次數。
- 比較不同排序法的操作次數。
- 用 merge sort 理解 $O(N \log N)$。
- 分治找答案。

可放進 challenge：

- Inversion count。

## 整體節奏

```text
L3：整理資料
L4：處理文字與封裝邏輯
L5：判斷暴力可不可行
L6：預處理回答很多查詢
L7：用遞迴枚舉可能性
L8：在有序或單調中快速搜尋
L9：排序後做正確選擇
L10：理解排序與分治原理
```

這個順序的重點是讓學生每課都學到一種可立即拿來解題的模式，再逐步補上效率分析、抽象技巧與演算法原理。
